# 뷰티그라운드 상품 이미지 작업 가이드라인 (기준: 제품 484 완성본)

> petitfee(Cafe24 몰) 상품을 뷰티그라운드에 등록할 때 **대표이미지·상세이미지를 순서대로, 전부, 선명하게** 넣기 위한 확정 절차.
> 검증 제품: **[484] [기획특가] 하이드로겔 아이패치 12종 1+1** — 앱 상세가 petitfee와 동일 순서로 완성됨.

## 1. 문제였던 원인 (진단 결과)
- **상세 뒤죽박죽**: 기존 scrape-product API가 상세를 넓은 `.xans-product-detail`(27장)에서 긁어 **갤러리 썸네일이 상세 앞에 섞이고 순서가 깨짐**.
- **대표 작아 보임**: 원본은 대부분 1000~2000px로 충분한데, 앱 CSS가 대표를 `max-h-[380px]`로, 상세를 `max-w-3xl(768px)`로 **작게 표시**한 것.
- **해상도 착시**: 폴더명이 `extra/small`이어도 실제 1000~2000px임. **폴더 치환(→/big/) 금지** — 파일명이 달라 404.

## 2. 이미지 추출 규칙 (★확정 최종 레시피)
| 항목 | 컨테이너(선택자) | 순서 |
|---|---|---|
| **대표 갤러리** | **`.keyImg`(대표 고해상) + `.xans-product-addimage`의 2번째부터** | keyImg가 첫 장 |
| **상세 이미지** | **`#prdDetail .cont` 내 `img` 만** | **문서순 그대로(중요)** |

**갤러리 로직(중복 방지 핵심):**
```
key = collect('.keyImg')              // 대표 1장(big, 최고해상)
add = collect('.xans-product-addimage')  // petitfee가 실제 보여주는 썸네일들(첫 장 = 대표의 small버전)
gallery = key.length && add.length ? [key[0], ...add.slice(1)] : (add.length ? add : collect('.xans-product-image'))
```
→ 결과 개수 = **petitfee가 화면에 보여주는 썸네일 개수와 정확히 일치**해야 정상.

**공통 필터(갤러리·상세 모두):**
- 제외 정규식(파일명/URL 기준): `img.echosting.cafe24.com | .gif | icon | btn | banner | logo | blank | spacer | sprite | /(top|bottom)\d+_ | txt_naver`. (배지/스페이서 `top0_reserve`·`top1_member`·`top2_gift`·`bottom1_delivery`가 여기서 걸림.)
- **★크기(naturalWidth) 기반 스페이서 판정 금지** — Cafe24 지연이미지는 로딩 전 `naturalWidth=1×1`(플레이스홀더)라 **진짜 상세컷을 스페이서로 오인해 다 걸러버린다**(실제로 낸 오류). URL/파일명 필터만 쓸 것.
- **src는 `ec-data-src` 최우선** — `ec-data-src` 있으면 그것(진짜 URL) 사용. **srcset로 덮어쓰지 말 것** — 지연이미지의 srcset은 플레이스홀더라 필터링돼 상세가 0이 된다(실제로 낸 오류).
- **옵션영역 제외**: `.xans-product-addproduct` `.xans-product-option` `.xans-product-relationproduct` `[class*="addProduct"]` 의 이미지는 갤러리에서 반드시 뺀다.
- 반드시 **Playwright로 끝까지 스크롤(mouse.wheel)** 후 수집. **무거운 신형 상세페이지는 `#prdDetail .cont`가 AJAX 지연로딩**이라 안 뜰 때가 있음 → `/upload/(NNEditor|product/...)` 이미지가 뜰 때까지 대기, 안 되면 **리로드 재시도(최대 4회)**.

## 2-1. ★반복하지 말 것 — 실제로 냈던 오류 5종 (배포/보고 전 매번 체크)
1. **페이지네이션 누락** — 목록을 1페이지만 보고 끝냄(아이패치 12개만 → 실제 18개). **모든 page + 모든 카테고리 탭** 순회. cate_no=24의 24는 카테고리번호(제품수 아님).
2. **상세 뒤죽박죽/갤러리 누수** — 상세를 넓은 `.xans-product-detail`서 긁어 갤러리가 상세 앞에 박힘. → **`#prdDetail .cont`만**.
3. **갤러리에 옵션 이미지** — 추가구성상품/구매옵션 썸네일이 갤러리에 섞임. → 옵션영역 제외.
4. **스페이서 누수** — `top0_reserve.jpg`(1000×6) 같은 얇은 줄이 상세에 들어감. → 자연크기 필터.
5. **대표이미지 중복(6 vs 5)** — `.xans-product-image`는 keyImg(big)+addimage(small대표)를 둘 다 담아 대표가 두 번 들어감. → 갤러리 로직(위) 사용. **개수가 petitfee 화면 썸네일 수와 같은지 반드시 대조.**
6. **지연이미지를 1×1로 오인해 상세=0** — naturalWidth 기반 스페이서 판정 + srcset 덮어쓰기 때문에 신형 상세페이지(포어패드·바디스프레이 등)의 진짜 상세컷을 다 버림. → **naturalWidth 판정 금지 + ec-data-src 최우선(srcset 금지)**. 상세=0 나오면 리로드 재시도.
7. **병렬 과부하로 상세 지연로딩 누락** — 병렬 워커 많으면 무거운 페이지 상세가 안 뜸. → 병렬 2 이하 + 상세 로딩 대기/재시도. 이미 잘 된 제품은 재실행 시 SKIP(덮어쓰기 방지).

### 배포/완료 보고 전 검증 체크리스트 (전수·실측)
- [ ] 갤러리 개수 == petitfee 화면 썸네일 개수 (중복·옵션·스페이서 0)
- [ ] 갤러리 ∩ 옵션영역 == 0
- [ ] 상세 첫 이미지가 `/product/(big|small|tiny|extra)` 아님(= 갤러리 누수 없음), 순서 == 원본
- [ ] 라이브에서 실측(개수·순서·크기) + 원본과 1:1 대조 (스크린샷)
