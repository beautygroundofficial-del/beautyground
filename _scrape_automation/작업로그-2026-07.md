## ⏳ 다음 이어서 할 작업 — 리뷰 기능 (진행중, 소스수정 전)
**목표**: "대표이미지 → 리뷰 → 상세이미지" 순서로 리뷰 섹션 배치(고객+관리자), 72개 리뷰 수집.

**현재까지 파악(조사만 함, 소스 미수정)**:
- 고객 상세(AppProductDetail)는 **이미 scraped_reviews를 렌더**함(ReviewMarquee=포토카드 띠, PhotoReviewShowcase). 리뷰 데이터만 채우면 대표↔상세 사이에 나옴. 지금은 72개 리뷰가 비어 안 보임.
- **관리자(파트너) 상세엔 리뷰 표시 없음** → 리뷰 섹션 새로 추가해야 함(대표/헤더 ↔ 상세 사이).
- **사용자 요청 레이아웃**: 대표 밑 리뷰 박스 **간격 여유**, 리뷰/상세 공간도 여유(padding/margin).

**리뷰 데이터 실측(중요)**:
- petitfee 리뷰 = Cafe24 게시판 board_no=4. 리스트는 원본HTML에 서버렌더(`.xans-product-review tr.xans-record-`, 페이지당 10, `/article/review/4/<no>/` 링크). 상세에 본문+`review-attachment-*.jpeg` 사진.
- **기존 `/api/scrape-reviews`(cheerio)는 petitfee에 0 반환** — 셀렉터 불일치(리스트는 raw HTML에 있으니 고치면 될 수도).
- **리뷰 대부분이 네이버페이 "만족/불만족" 평점만**(본문·사진 없음). 글/사진 리뷰는 드묾.
- Playwright 스크래퍼 `pwtest/scrape-reviews-pw.mjs` 작성함: rating(만족5/불만족2/보통3)·author·date는 잘 나옴, **사진 추출은 아직 미완**(review-attachment 필터 보강 필요).

**★확정 디자인(사용자 지시, A/B/C 폐기)**: 리뷰 섹션을 **간단하게** — 복잡한 개별 리뷰카드/텍스트 불필요.
- 상단: **전체 리뷰 (N)** + **★평점/5.0** + **리뷰수 N** (Snapreview 위젯 스타일이 참고. petitfee는 Snapreview 안 씀 = 디자인만 참고).
- 하단: **작은 리뷰 사진 썸네일** 몇 개.
- 필요한 데이터 = ①리뷰수(petitfee 게시판 "후기 N", 예 product17=340) ②평균평점(만족5/불만족2/보통3 로 계산, 대부분 만족이라 ~4.7~4.9) ③리뷰사진 썸네일(review-attachment).

**이어서 할 순서**:
1. `scrape-reviews-pw.mjs`에서 **리뷰수 + 평균평점 + 사진썸네일**만 추출하게 단순화(본문 텍스트 불필요). **사진추출(review-attachment) 보강 필요** — 현재 미완(사진0 나옴). 사진 있는 리뷰(ico_attach 행)의 상세에서 `img[src*=review-attachment]` 잡기.
2. 72개 배치 수집 → 저장. (scraped_reviews에 사진배열+집계 저장하거나, 집계는 별도 컬럼/계산)
3. **간단 리뷰 위젯 컴포넌트** 새로 만들어 대표→리뷰→상세 위치에 배치(고객+관리자), 대표 밑 간격·리뷰/상세 공간 **여유(padding/margin)** 있게.
4. 고객+관리자 라이브 검증.
- 참고: AppProductDetail은 사용자 미커밋 모바일작업 있어 **선별커밋(HEAD복원→편집→커밋→백업복원)** 필요. 백업 최신=`scratchpad/AppProductDetail.v3.bak`.
- 참고: 기존 ReviewMarquee/PhotoReviewShowcase(개별 리뷰카드)는 이 간단 디자인과 다름 — 새 간단 위젯으로 대체하거나 상단에 얹기.
